Вот стек, который я бы порекомендовал для полного переписывания вашего проекта, чтобы он был современным, надёжным и приятным в разработке:

Бэкенд: Go + Gin (как API-сервер), GORM (для БД), Viper (для конфигурации), OpenAPI/Swagger (для документирования API).

Фронтенд: React.js + TypeScript (для надёжности), собранный с помощью Vite (для скорости разработки), со стилями на Tailwind CSS.

Связь: TanStack Query для "умных" запросов данных с фронтенда на бэкенд.

Начать можно с создания нового фронтенд-проекта (npm create vite@latest), а затем постепенно переносить логику, создавая новые API-эндпоинты в Go и "общаясь" с ними из вашего нового React-приложения.
